# GNSS HAL Mock Implementation

This directory contains the custom GNSS HAL implementation (`android.hardware.gnss@2.1-impl.so`) that replaces the vendor HAL via KernelSU.

## Compilation

Because this HAL implements Android HIDL interfaces (`android.hardware.gnss@2.1::IGnss`), it depends on internal Android system libraries (`libhidlbase`, `libutils`) and generated HIDL headers. **It cannot be compiled directly with the Android NDK in Android Studio.**

To compile this HAL:
1. Place this directory into an AOSP source tree (e.g., `hardware/interfaces/gnss/mock/`).
2. Run `m android.hardware.gnss@2.1-impl-mock`.
3. Rename the output `.so` to `android.hardware.gnss@2.1-impl.so`.
4. Copy the compiled `.so` (for `arm64-v8a` and `armeabi-v7a`) into the `app/src/main/assets/ksu_module/system/vendor/lib64/hw/` directory of this Android Studio project.

## Architecture

This mock HAL bypasses the real GNSS hardware entirely.
- `start()`: Opens `/dev/gnss_spoof` (a named pipe/character device created by the daemon).
- `readLoop()`: Continuously reads NMEA sentences generated by `gnss_spoof_daemon`.
- It feeds these fake NMEA strings and coordinates up to the Android framework (`system_server`).

By replacing the vendor HAL, the system is completely unaware that it is receiving mocked data, even when flight mode is off and real hardware is powered on.
